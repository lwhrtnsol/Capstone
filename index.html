Professional Self-Assessment

I am completing a Bachelor of Science in Computer Science with a concentration in Project Management at Southern New Hampshire University. I entered the program with the goal of moving from automotive production into a technical or project-focused career. That goal has remained consistent, but the program helped me better understand how software development, data systems, cybersecurity, testing, and project coordination work together to support reliable business operations.

My experience at Toyota has shaped the way I approach technology and problem solving. Working in manufacturing has shown me the importance of quality, safety, standardized work, communication, and continuous improvement. I have learned that technical solutions are more valuable when the people using them understand the process, trust the information, and can depend on the system during day-to-day operations. This perspective has helped me connect computer science concepts to real workplace needs and stakeholder expectations.

Throughout the program, I developed experience with collaboration, communication, and decision-making. In CS 250, I worked with Agile and Scrum concepts such as user stories, sprint planning, reviews, and retrospectives. Those activities helped me understand how developers, project managers, and stakeholders use communication to clarify requirements and improve a product over time. My work experience also reinforced that teams make better decisions when information is clear, risks are discussed early, and improvements are tied to the needs of the people affected by them.

The program strengthened my technical skills across software engineering, algorithms, databases, and security. I learned to design applications using organized structures, validate input, test functionality, manage data, and evaluate trade-offs. My work with algorithms and data structures helped me understand how data is processed efficiently and how logic affects the reliability of a solution. My database coursework strengthened my understanding of relationships, constraints, migrations, and secure handling of stored information. Security coursework also taught me to consider authentication, configuration, input validation, privacy, and potential vulnerabilities before software is deployed.

My three ePortfolio artifacts bring these skills together. Travlr Getaways represents software design and engineering through full-stack validation, API error handling, secure configuration, and dynamic trip rendering. The SSL Server represents algorithmic thinking through SHA-256 hashing, byte-array processing, validation, and structured responses. ChrisMitchellInventory represents database design through SQLite relationships, migration logic, validation, and user-specific inventory records. Together, these artifacts provide evidence of my ability to use current tools, evaluate computing solutions, apply security practices, communicate technical decisions, and support organizational goals.

My next career step is to move into a technical, project-focused, or operations support role where I can contribute to software, data, process improvement, and business systems. I plan to continue developing my technical skills, gain project experience, and pursue professional certifications such as CAPM. This ePortfolio represents both the work I have completed during the program and the direction I intend to take in my career.

Software Design and Engineering: Travlr Getaways

Original Artifact

Travlr Getaways is a full-stack travel booking application originally developed in CS 465: Full Stack Development. It includes a Node.js and Express public website, an Angular administrative interface, a REST API, and MongoDB data storage.

Enhancements Implemented
  • Added schema-level validation and clearer trip-data error messages.
  • Improved API controller error handling for invalid requests, duplicates, and missing records.
  • Moved MongoDB and JWT configuration values into environment variables.
  • Added a Handlebars-compatible error page and improved dynamic trip rendering.
Skills and Outcomes Demonstrated

This artifact demonstrates full-stack software design, validation, secure configuration, maintainability, API reliability, and communication of technical decisions. It supports computing practices, security, design trade-offs, and professional communication outcomes.

Algorithms and Data Structures: Project Two SSL Server

Original Artifact

Project Two SSL Server is a Java Spring Boot application originally developed in CS 305: Software Security. The application uses HTTPS and provides a /hash endpoint that generates a SHA-256 checksum from user-provided or default input.

Enhancements Implemented
  • Moved hashing and input validation into a separate HashService class.
  • Added HashResponse to return organized JSON instead of manually generated HTML.
  • Added blank-input and maximum-length validation.
  • Improved byte-to-hex conversion with a character array and bitwise operations.
  • Moved the SSL keystore password into an environment variable.
Skills and Outcomes Demonstrated

This artifact demonstrates algorithmic principles, byte-array processing, SHA-256 hashing, input validation, structured response design, testing, and secure application configuration. It supports algorithmic design, current computing practices, security, and communication outcomes.

Databases: ChrisMitchellInventory Android Application

Original Artifact

ChrisMitchellInventory is an Android inventory application originally developed for a mobile programming course. It uses SQLite to store user accounts, password information, and inventory records.

Enhancements Implemented
  • Added a user_id relationship between users and inventory items.
  • Enabled SQLite foreign key constraints and added a quantity CHECK constraint.
  • Replaced the drop-and-recreate upgrade approach with a data-preserving migration.
  • Added user-specific create, read, update, and delete repository methods.
  • Improved validation, password handling, and user account logic.
Skills and Outcomes Demonstrated

This artifact demonstrates relational database design, foreign keys, migrations, data validation, repository patterns, secure password handling, and user-specific data protection. It supports computing practices, security, database design, technical communication, and organizational decision-making outcomes.
